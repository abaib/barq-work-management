User Role Definition Clarification

Role-mapping glossary resolving inconsistencies between primary and specialized roles

The Challenge: The document defines 8 primary user roles, but detailed RBAC matrices introduce additional specialized roles (HR Manager, Legal Officer, Accountant, Customer Service Rep) that don't appear in the primary list. This creates ambiguity for RBAC configuration, licensing, and user assignment.

8 Primary Roles

  • System Administrator
  • Project Manager
  • Field Technician
  • QC Inspector
  • HSE Officer
  • Warehouse Staff
  • Finance Manager
  • Viewer/Analyst

Specialized Role Mapping

Specialized Role Maps to Primary Role Module-Specific Permissions License Type
HR Manager System Administrator Employee Management: Full CRUD on employee records, leave approval, performance reviews Admin License
Legal Officer Finance Manager Contract Management: Full CRUD on contracts, legal review workflow, compliance tracking Standard License
Accountant Finance Manager Financial Management: Full CRUD on invoices, expenses, budgets; Limited access to contracts Standard License
Customer Service Rep Viewer/Analyst Service Request Management: Create/Edit service requests, view customer data, no financial access Standard License
Clarification Rules: No new user licenses required. Specialized roles inherit base permissions from primary role, then add module-specific permissions. A single user can have one primary role + multiple specialized role permissions (e.g., "John Doe" = Finance Manager + Accountant + Legal Officer).
